# Plan 09-03: SEO Localization & Final Polish

## Objective
Finalize the internationalization phase by ensuring search engines can correctly index both language versions of the site and that global utilities (analytics, SEO metadata) are fully locale-aware.

## Tasks

### 1. SEO & Metadata Localization
- [ ] **Task 1.1: Localized Sitemap**
  - <read_first>src/app/sitemap.ts</read_first>
  - <action>Update `src/app/sitemap.ts` to generate URLs for both `en` and `bn` locales. Include `alternates` with `languages` mapping to ensure Google understands the relationship between pages.</action>
  - <acceptance_criteria>Building the project generates a sitemap containing both `/features` and `/bn/features` (or equivalent structure).</acceptance_criteria>
- [ ] **Task 1.2: Hreflang Tags in Layout**
  - <read_first>src/app/[locale]/layout.tsx</read_first>
  - <action>Update the metadata in `src/app/[locale]/layout.tsx` to include `alternates.languages` for the current page, ensuring hreflang tags are injected into the HTML head.</action>
  - <acceptance_criteria>Viewing the source of any page shows `<link rel="alternate" hreflang="bn" ...>` and `<link rel="alternate" hreflang="en" ...>`.</acceptance_criteria>

### 2. Analytics & Utility Polish
- [ ] **Task 2.1: Locale-Aware Analytics**

  - <acceptance_criteria>Analytics events triggered on `/bn/*` pages include a `locale: bn` property.</acceptance_criteria>
