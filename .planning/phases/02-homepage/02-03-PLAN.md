### Task 2: Verify responsive layout across all breakpoints

<read_first>
- `src/app/page.tsx` — composed homepage from Task 1
- `src/app/globals.css` — all section CSS classes and responsive overrides
- `woobooster-v2.html` lines 382-405 — Responsive breakpoint definitions
- `src/components/sections/HeroSection.tsx` — check responsive classes
- `src/components/sections/FeaturesBento.tsx` — check grid responsive behavior
- `src/components/sections/BDSection.tsx` — check 2-column layout responsive
</read_first>

<action>
Verify and fix responsive behavior across all section components. The HTML reference defines two breakpoints: 960px (tablet) and 640px (mobile).

**Check each section for correct responsive classes:**

1. **HeroSection** — hero grid should be `grid-cols-1 lg:grid-cols-2` with dashboard mockup `hidden lg:block`. If the component uses Tailwind `lg:` breakpoint (1024px), confirm this is acceptable as a close match to 960px. If it uses a custom breakpoint, verify it's defined.

2. **FeaturesBento** — bento grid uses CSS classes from globals.css: `grid-cols-3` → `grid-cols-2` at 960px → `grid-cols-1` at 640px. Verify the `@media` queries in globals.css handle this. The `.bc.w2` should be `col-span-2` on tablet, `col-span-1` on mobile.

3. **TrustBar** — flex-wrap should handle narrow widths. Stats should wrap to multiple lines on mobile.

4. **VideoSection** — video player should scale fluidly with `w-full`. Aspect ratio preserved by `aspect-video`.

5. **BDSection** — `.bd-layout` grid should be `grid-cols-1 lg:grid-cols-2`. On mobile, courier cards stack vertically.

6. **HowItWorks** — `.steps-grid` should be `grid-cols-1 lg:grid-cols-3`. On mobile, single column.

7. **Testimonials** — `.tgrid` should be `grid-cols-1 lg:grid-cols-3`. On mobile, single column.

8. **CTASection** — `.cta-wrap` padding reduces on mobile: `py-20 px-[60px]` → `py-12 px-7` at 640px.

**If any responsive classes are missing or incorrect**, add them to either the component's JSX classes or to `globals.css` responsive overrides. Prefer Tailwind responsive prefixes (`sm:`, `lg:`) in JSX when possible, and `@media` queries in globals.css for custom CSS classes.

**Specific check:** The HTML reference uses `max-width:960px` as the tablet breakpoint, but Tailwind's `lg:` is 1024px. This 44px difference is acceptable for this marketing site — no custom breakpoint needed. Document this as a known minor deviation from the HTML reference.
</action>

<acceptance_criteria>
- HeroSection dashboard is hidden below `lg` breakpoint
- FeaturesBento grid is 3 columns on desktop, 2 on tablet, 1 on mobile
- BDSection layout is 2 columns on desktop, 1 on mobile
- HowItWorks grid is 3 columns on desktop, 1 on mobile
- Testimonials grid is 3 columns on desktop, 1 on mobile
- CTASection padding reduces on mobile
- All sections have appropriate padding that doesn't overflow on mobile
- `pnpm build` succeeds
