## Design System

| Property | Value |

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding |
| sm | 8px | Compact element spacing, inline gaps |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, metric card grid gap |
| xl | 32px | Layout gaps, page-level margins |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Metric card icon container: 48px x 48px (12 in Tailwind) — matches EcommerceMetrics pattern
- Badge pill padding: `px-2.5 py-0.5` (10px/2px) — fixed by Badge component
- Notification bell button: 44px x 44px (h-11 w-11) — touch target minimum
- Mobile sidebar toggle: 40px x 40px minimum touch target

## Page-by-Page Visual Contracts

### 1. Dashboard Overview (`(portal)/dashboard/page.tsx`)

**Layout:** PageBreadCrumb + 4-column metric card grid + recent activity table

**Metric Cards** (grid: 1 col mobile, 2 col sm, 4 col lg):
| Card | Icon (lucide) | Icon BG | Value Source | Label |
|------|---------------|---------|-------------|-------|
| Active Licenses | Shield | `bg-green-lt` | `count(*)` where status=active | "Active Licenses" |
| Expiring Soon | Clock | `bg-orange-lt` | `count(*)` where active + expires < 30d | "Expiring Soon" |
| Recent Downloads | Download | `bg-accent-light` | `count(*)` from downloads | "Downloads" |
| Open Tickets | Ticket | `bg-orange-lt` | `count(*)` where status IN (open, in_progress) | "Open Tickets" |

Each card follows EcommerceMetrics pattern:
- Container: `rounded-2xl border border-gray-200 bg-white p-5 dark:border-gray-800 dark:bg-white/[0.03] md:p-6`
- Icon container: `w-12 h-12 rounded-xl` with semantic color background
- Label: `text-sm text-gray-500 dark:text-gray-400`
- Value: `text-title-sm font-bold text-gray-800 dark:text-white/90`

**Empty state (D-04):** All four cards show `0`. Below cards, show a welcome message:
- Heading: `text-xl font-semibold` "Welcome to ConversionFlow"
- Body: `text-sm text-gray-500` "Your dashboard is ready. Once you purchase a license or open a support ticket, your activity will appear here."

**Recent Activity Table:** Not required in Phase 3 per CONTEXT.md (only metric cards + welcome). If added later, use Table component with columns: Activity, Date, Status.

### 2. License List (`(portal)/licenses/page.tsx`)

**Layout:** PageBreadCrumb + ComponentCard wrapping Table

**Table columns:**
| Column | Width | Content | Component |
|--------|-------|---------|-----------|
| License Key | 25% | Masked key `CF-****-****-ABCD` + Copy button | LicenseKeyCopy client component |
| Plan | 15% | Plan text (e.g., "Professional") | Plain text |
| Status | 15% | Status badge | Badge (color per map above) |
| Activations | 15% | `currentActivations / maxActivations` | Plain text |
| Expires | 15% | Formatted date or "Never" | Plain text (date-fns format) |
| Actions | 15% | "View Details" button | `btn-outline` style button |

**Key masking pattern (D-06):**
- Display: `${key.slice(0, 2)}-${"*".repeat(4)}-${"*".repeat(4)}-${key.slice(-4)}`
- Copy button: 20px icon, inline next to masked key
- On copy: icon changes to checkmark for 2 seconds, button text shows "Copied!"

**Mobile:** Table scrolls horizontally within card. No card-view transformation.

### 3. License Detail (`(portal)/licenses/[id]/page.tsx` OR modal)

**Decision D-07:** Slide-in panel from right side.

**Panel layout:**
- Overlay: `fixed inset-0 bg-gray-400/50 backdrop-blur-[32px]` (matches Modal backdrop)
- Panel: `fixed right-0 top-0 h-full w-full max-w-lg bg-white dark:bg-gray-900 overflow-y-auto shadow-theme-lg`
- Close button: top-right `X` button matching Modal close button pattern
- Animation: slide in from right (translate-x 100% to 0, 300ms ease-out)

**Panel content sections:**
1. **Header:** License key (full, with copy button) + status badge
2. **Details grid (2-column):** Plan, Product, Created date, Expiry date
3. **Activation Domains:** List of domains with `currentActivations / maxActivations` count. Each domain row shows domain name + deactivate button (destructive).
4. **Empty domains:** "No domains activated yet." text

**Destructive action — Deactivate Domain:**
- Inline confirmation: clicking "Deactivate" replaces button text with "Confirm?" (red), clicking again fires action. Auto-reverts after 3 seconds.

## Interaction Patterns

### Copy to Clipboard (License Key)
- Client component: `LicenseKeyCopy`
- Click triggers `navigator.clipboard.writeText(fullKey)`
- Icon swaps from Copy to Check for 2000ms
- Accessible: `aria-label="Copy license key"`, `role="button"`

### Modal Open/Close
- Uses existing Modal component (escape key + backdrop click to close)
- Body scroll lock handled by Modal's `overflow: hidden` on body
- Animation: fade-in backdrop + scale-in content (already in Modal component)

### Slide-in Panel (License Detail)
- Client component triggers panel visibility
- Panel slides from right: `transform: translateX(100%)` -> `translateX(0)`, 300ms ease-out
- Backdrop overlay same as Modal pattern
- Close via X button, Escape key, or backdrop click

### Expand/Collapse (Changelog)
- Client component manages `isExpanded` state per version row
- Expanded content uses `overflow-hidden` with max-height transition
- Chevron icon rotates 180 degrees on expand

### Form Submission (Account, Create Ticket, Reply)
- Server actions handle all mutations
- Pending state: button shows loading spinner, disabled
- Success: inline success message, auto-hide after 3 seconds
- Error: inline error message below form, stays until dismissed

### Notification Mark as Read
- Click notification item -> server action `markNotificationRead(id)`
- Remove left border indicator (unread marker)
- Decrement unread count badge
- Navigate to entity URL from `data` JSONB field

### File Upload (Ticket Attachments)
- Button-triggered file picker (not drag-and-drop, per discretion)
- Accepted types: image/png, image/jpeg, image/webp, application/pdf
- Max file size: 10MB
- Selected files shown as chips with file name + remove (X) button
- Upload happens on form submit via server action

## Mobile Responsive Behavior

### Tables
All tables use horizontal scroll within their card container on viewports < 768px. No card-view transformation. Pattern:
```
<div className="overflow-x-auto">
  <Table className="min-w-[600px]">...</Table>
</div>
```

### Metric Cards
Grid collapses: 4 columns on lg, 2 columns on sm, 1 column on mobile.

### Slide-in Panel (License Detail)
Panel takes full width on mobile (`w-full`), max 512px on desktop (`max-w-lg`).

### Ticket Conversation
Message bubbles widen to `max-w-[85%]` on mobile (from `max-w-[70%]` on desktop).

### Account Settings
Profile fields: 2-column grid collapses to 1-column on mobile.
