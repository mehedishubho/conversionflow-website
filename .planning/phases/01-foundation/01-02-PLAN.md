**Step 1: Create src/lib/auth.ts -- Better Auth server configuration.**

This is the core auth configuration file. It must include:

```typescript
import { betterAuth } from "better-auth";
import { drizzleAdapter } from "better-auth/adapters/drizzle";
import { admin, twoFactor } from "better-auth/plugins";
import { db } from "@/lib/db";
import * as schema from "@/lib/db/schema";
import { kvGet, kvSet, kvDelete } from "@/lib/redis";

export const auth = betterAuth({
  database: drizzleAdapter(db, {
    provider: "pg",
    schema,
  }),
  emailAndPassword: {
    enabled: true,
    minPasswordLength: 8, // D-07: minimum 8 characters
  },
  session: {
    expiresIn: 60 * 60 * 24 * 30, // D-10: 30 days in seconds
    updateAge: 60 * 60 * 24, // Refresh session every 24 hours
    cookieCache: {
      enabled: true,
      maxAge: 60 * 5, // Cache session in cookie for 5 minutes
    },
  },
  user: {
    additionalFields: {
      role: {
        type: "string",
        defaultValue: "customer",
        input: false, // Not settable during registration -- prevents mass assignment
      },
      phone: {
        type: "string",
        required: true, // D-05: phone required at registration
      },
      centralUserId: {
        type: "string",
        required: false,
        input: false, // Set by central API integration, not by users
      },
    },
  },
  plugins: [
    admin(),
    twoFactor(),
  ],
  secondaryStorage: {
    get: async (sessionId) => {
      const data = await kvGet(`session:${sessionId}`);
      return data ? JSON.parse(data) : null;
    },
    set: async (sessionId, data, ttl) => {
      await kvSet(`session:${sessionId}`, JSON.stringify(data), ttl);
    },
    delete: async (sessionId) => {
      await kvDelete(`session:${sessionId}`);
    },
  },
  databaseHooks: {
    user: {
      create: {
        after: async (user) => {
          // Audit log: user registration
          // Will be fully wired in Task 2
        },
      },
    },
    session: {
      create: {
        after: async (session) => {
          // Audit log: user login
          // Will be fully wired in Task 2
        },
      },
    },
  },
});

export type Auth = typeof auth;
```

**Key design decisions:**
- `input: false` on `role` field prevents users from setting their own role during registration (mass assignment protection, D-02/AUTH-02)
- `secondaryStorage` always provided (uses in-memory fallback from Plan 01 when Redis unavailable) per D-14
- Session duration 30 days per D-10
- Min password length 8 per D-07
- `admin()` plugin enables admin user management
- `twoFactor()` plugin enables TOTP 2FA readiness per AUTH-06

**Step 1: Create src/lib/audit.ts -- Audit logging helper.**

```typescript
import { db } from "@/lib/db";
import { auditLogs } from "@/lib/db/schema";

type AuditAction =
  | "user.registered"
  | "user.login"
  | "user.logout"
  | "user.password_reset"
  | "user.email_verified"
  | "user.role_changed"
  | "user.banned"
  | "user.activated"
  | "order.created"
  | "order.status_changed"
  | "license.created"
  | "license.status_changed"
  | "ticket.created"
  | "ticket.updated"
  | "admin.setup_completed"
  | string;

interface AuditLogEntry {
  actorId?: string;
  actorRole?: string;
  action: AuditAction;
  targetType?: string;
  targetId?: string;
  details?: Record<string, unknown>;
  ipAddress?: string;
}

export async function createAuditLog(entry: AuditLogEntry): Promise<void> {
  try {
    await db.insert(auditLogs).values({
      actorId: entry.actorId ?? null,
      actorRole: entry.actorRole ?? null,
      action: entry.action,
      targetType: entry.targetType ?? null,
      targetId: entry.targetId ?? null,
      details: entry.details ?? null,
      ipAddress: entry.ipAddress ?? null,
    });
  } catch (error) {
    // Audit logging should never break the main flow
    console.error("[Audit] Failed to create audit log:", error);
  }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> /api/auth/* | Untrusted HTTP requests to Better Auth API routes |
| Better Auth -> PostgreSQL | Auth data persistence with parameterized queries |
| Better Auth -> Redis | Session storage via secondaryStorage interface |
| App -> Auth config | Server-side auth configuration, no client exposure |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | T (Tampering) | user.role field | mitigate | `input: false` on role additionalField prevents mass assignment during registration. Only server-side code or admin API can change roles. |
| T-01-06 | S (Spoofing) | Session tokens | mitigate | Better Auth generates cryptographically secure session tokens. Stored in HttpOnly cookies. Redis secondaryStorage validates session existence on every request. |
| T-01-07 | E (Elevation) | RBAC bypass | mitigate | admin() plugin provides server-side role checks. Role checks happen in page components (server-side), not just in proxy. |
| T-01-08 | I (Information Disclosure) | Audit logs | accept | Audit logs contain actor ID, action, target -- no passwords or sensitive tokens. Access restricted to admin routes. |
| T-01-09 | D (Denial of Service) | Redis unavailable | mitigate | secondaryStorage uses kvGet/kvSet/kvDelete which gracefully fall back to in-memory store when Redis is down. |
</threat_model>
