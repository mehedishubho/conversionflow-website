**Step 1: Rewrite src/proxy.ts with route protection.**

The proxy must:
1. Check for auth session cookie on protected routes (portal + admin)
2. Redirect unauthenticated users to /login
3. Redirect authenticated users away from auth pages (login, register) to appropriate dashboard
4. Skip i18n routing for all non-marketing routes (auth, portal, admin, setup)
5. Preserve existing i18n routing for marketing pages

```typescript
import createMiddleware from 'next-intl/middleware';
import { routing } from './i18n/routing';
import type { NextRequest } from 'next/server';
import { NextResponse } from 'next/server';

const handleI18nRouting = createMiddleware(routing);

export function proxy(request: NextRequest) {
  const { pathname } = request.nextUrl;

  // Skip static files, API routes, and _next
  if (
    pathname.includes('.') ||
    pathname.startsWith('/api') ||
    pathname.startsWith('/_next')
  ) {
    return;
  }

  // Define route categories
  const isAuthPage = pathname === '/login' ||
    pathname === '/register' ||
    pathname === '/verify-email' ||
    pathname === '/forgot-password' ||
    pathname === '/reset-password';

  const isPortalRoute = pathname.startsWith('/dashboard') ||
    pathname.startsWith('/licenses') ||
    pathname.startsWith('/billing') ||
    pathname.startsWith('/downloads') ||
    pathname.startsWith('/tickets') ||
    pathname.startsWith('/notifications') ||
    pathname.startsWith('/checkout') ||
    pathname.startsWith('/account');

  const isAdminRoute = pathname.startsWith('/admin');

  const isNonMarketingRoute = isAuthPage || isPortalRoute || isAdminRoute;

  const sessionCookie = request.cookies.get('better-auth.session_token');

  // Protected routes: redirect to login if no session
  if ((isPortalRoute || isAdminRoute) && !sessionCookie) {
    const loginUrl = new URL('/login', request.url);
    loginUrl.searchParams.set('callbackUrl', pathname);
    return NextResponse.redirect(loginUrl);
  }

  // Auth pages: redirect logged-in users to dashboard
  if (isAuthPage && sessionCookie) {
    return NextResponse.redirect(new URL('/dashboard', request.url));
  }

  // Non-marketing routes: pass through without i18n
  if (isNonMarketingRoute) {
    return NextResponse.next();
  }

  // Marketing routes: apply i18n routing
  return handleI18nRouting(request);
}

export const config = {
  matcher: [
    '/',
    '/(bn|en)/:path*',
    '/((?!api|_next|_vercel|.*\\..*).*)'
  ]
};
```

**Important notes:**
- The cookie name `better-auth.session_token` needs verification during implementation. Check what Better Auth actually names its session cookie (assumption A7 in RESEARCH.md).
- The proxy does NOT check roles -- that happens server-side in page components. This is by design: Edge runtime cannot query the database.
- The `callbackUrl` search param preserves the intended destination so login can redirect back after auth.

**Step 1: Create src/app/(admin)/setup/page.tsx -- First admin setup wizard.**

This page implements D-11 (setup wizard as primary method for creating first admin).

The page is a server component that:
1. Checks if any super_admin exists in the database
2. If super_admin exists: returns a 404-like response (redirect to /login or render "Setup already completed")
3. If no super_admin exists: renders a setup form

The setup form (client component embedded in the page):
- Fields: full name, email, password, confirm password
- Optional: ADMIN_SETUP_TOKEN from env vars (if set, require it; if not, skip)
- Submit creates a user with role super_admin
- After success, auto-login and redirect to /admin/dashboard

Implementation structure:

```typescript
// src/app/(admin)/setup/page.tsx
import { db } from "@/lib/db";
import { eq, sql } from "drizzle-orm";
import { user } from "@/lib/db/schema";
import { redirect } from "next/navigation";
import SetupForm from "./SetupForm";

export default async function SetupPage() {
  // Check if any super_admin already exists
  const adminCount = await db
    .select({ count: sql<number>`count(*)` })
    .from(user)
    .where(eq(user.role, "super_admin"));

  if (adminCount[0].count > 0) {
    // Setup already completed -- redirect to login
    redirect("/login");
  }

  return (
    <div className="flex items-center justify-center min-h-screen">
      <div className="w-full max-w-md px-6">
        <div className="text-center mb-8">
          <h1 className="text-2xl font-bold mb-2">Initial Admin Setup</h1>
          <p className="text-[--text2]">
            Create the first super admin account for ConversionFlow
          </p>
        </div>
        <SetupForm />
      </div>
    </div>
  );
}
```

```typescript
"use server";

import { db } from "@/lib/db";
import { user } from "@/lib/db/schema";
import { eq, sql } from "drizzle-orm";

export async function completeSetup(userId: string) {
  // Double-check no super_admin exists (race condition protection)
  const adminCount = await db
    .select({ count: sql<number>`count(*)` })
    .from(user)
    .where(eq(user.role, "super_admin"));

  if (adminCount[0].count > 0) {
    return { error: "Setup already completed. An admin account exists." };
  }

  // Set role to super_admin
  await db
    .update(user)
    .set({ role: "super_admin" })
    .where(eq(user.id, userId));

  return { success: true };
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User -> Proxy (Edge) | Untrusted request hitting route protection layer |
| Proxy -> Page component | Session cookie passed but role NOT checked at edge |
| Page -> Auth API | Server-side session + role verification |
| User -> /admin/setup | Unauthenticated setup route (before admin exists) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-15 | E (Elevation) | Customer accessing /admin routes | mitigate | Server-side role check in every admin page component. Proxy only checks session existence. Role check at page level prevents privilege escalation. |
| T-01-16 | T (Tampering) | /admin/setup abuse | mitigate | Server-side check: count of super_admin users. If > 0, redirect away. Double-check in server action for race conditions. Optional ADMIN_SETUP_TOKEN env var for additional protection. |
| T-01-17 | S (Spoofing) | Session cookie forgery | mitigate | Better Auth signs session tokens cryptographically. Redis secondaryStorage validates session existence. HttpOnly cookies prevent JS access. |
| T-01-18 | T (Tampering) | callbackUrl open redirect | mitigate | Only allow relative URLs in callbackUrl parameter. Reject absolute URLs or URLs with different origins. |
| T-01-19 | I (Information Disclosure) | Error messages revealing system info | accept | Generic error messages on auth pages. No stack traces in production. |
</threat_model>
