## Decisions Made
- **Custom plugin for lockout:** The top-level `hooks` field in `betterAuth()` only accepts a single `AuthMiddleware`, not arrays. The `{ matcher, handler }` pattern is only available inside plugin hooks. So account lockout was implemented as a custom Better Auth plugin.
- **Client method names:** Better Auth client method names are derived from `operationId` in endpoint definitions. The actual names are `requestPasswordReset` (not `forgotPassword`) and `sendVerificationEmail` (not `emailVerification`).
- **pnpm override for better-call:** `@better-auth/cli@1.4.21` pulls `@better-auth/core@1.4.21` which requires `better-call@1.1.8`. The main `better-auth@1.6.11` needs `better-call@1.3.5` for `kAPIErrorHeaderSymbol`. Added `pnpm.overrides.better-call: "^1.3.5"` to resolve.
- **No session table:** Better Auth CLI did not generate a `session` table because `secondaryStorage` is configured. Sessions are stored in Redis via kvGet/kvSet/kvDelete. This is correct behavior per D-14.
