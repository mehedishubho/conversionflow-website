# Plan 09-01: Internationalization Foundation & UI

## Objective
Implement the core infrastructure for bilingual support (English & Bengali) using `next-intl` and the Next.js 16 `proxy.ts` pattern. Migrate the current `src/app` structure to a locale-prefixed routing system and enable global translation for key UI components.

## Tasks

### 1. Infrastructure & Setup
- [ ] **Task 1.1: Install next-intl**
  - Run `pnpm add next-intl` to add the internationalization framework.
- [ ] **Task 1.2: Configure next-intl Routing**
  - Create `src/i18n/routing.ts` with locales `['en', 'bn']`, default locale `bn`, and `localePrefix: 'as-needed'`.
  - Create `src/i18n/request.ts` to load JSON messages from `src/messages/`.
- [ ] **Task 1.3: Implement Next.js 16 Proxy**
  - Create `src/proxy.ts` exporting a named `proxy` function.
  - Integrate `next-intl/middleware` into `proxy.ts` to handle locale detection and URL rewriting.
  - Ensure the matcher excludes `sitemap.xml`, `robots.txt`, `api/`, and static assets.

### 2. Structural Migration
- [ ] **Task 2.1: Route Relocation**
  - Create `src/app/[locale]` directory.
  - Move existing pages (Home, Features, Pricing, Changelog, Support, Legal routes) into `src/app/[locale]/`.
  - Maintain `src/app/layout.tsx` as a shared entry point if possible, or migrate it to `src/app/[locale]/layout.tsx`.
- [ ] **Task 2.2: Localized Root Layout**
  - Update `src/app/[locale]/layout.tsx` to receive the `locale` prop.
  - Wrap children with `NextIntlClientProvider`.
  - Apply `Noto_Sans_Bengali` font globally to the `body` when `locale === 'bn'`.

### 3. UI Localization
- [ ] **Task 3.1: Language Switcher (Navbar)**
  - Create `src/components/layout/LanguageToggle.tsx`.
  - Implement an `EN / বাং` toggle using `usePathname` and `useRouter` from `next-intl/navigation`.
  - Integrate into `src/components/layout/Navbar.tsx`.
- [ ] **Task 3.2: Footer Selector**
  - Add a language selection component to `src/components/layout/Footer.tsx`.
- [ ] **Task 3.3: String Extraction**
  - Refactor `HeroSection`, `Navbar`, and `Footer` to use `useTranslations()` instead of hardcoded strings.

### 4. Content Localization (Docs)
- [ ] **Task 4.1: Localized MDX Logic**
  - Update `src/lib/mdx.ts` to accept a `locale` parameter.
  - Modify fetching logic to prioritize `.bn.mdx` for documentation when the locale is `bn`.
  - Ensure English fallback for Blog and missing Doc translations.

## Verification Plan

### Automated Tests
- [ ] `pnpm build` must pass with the new routing structure.
- [ ] Verify that `/sitemap.xml` and `/robots.txt` are still accessible.

### Manual Verification
- [ ] Verify that `localhost:3000/` serves Bengali content.
- [ ] Verify that `localhost:3000/en` serves English content.
- [ ] Verify that `localhost:3000/` uses the `Noto Sans Bengali` font.
- [ ] Verify that switching languages preserves the current route.
- [ ] Verify that `/docs` loads the correct language version of a document.
