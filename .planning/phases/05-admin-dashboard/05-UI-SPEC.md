## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, table cell inner spacing |
| sm | 8px | Compact element spacing, gap between filter tabs |
| md | 16px | Default element spacing, card padding (px-4 py-3), table cell padding |
| lg | 24px | Section padding, card body padding (p-5/p-6) |
| xl | 32px | Layout gaps, grid column gaps between KPI cards |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- 12px (3x4) used for icon container size (`w-12 h-12` for KPI card icons)
- 40px (10x4) used for notification icon containers (`w-10 h-10`)
- 44px touch targets for action buttons in mobile sidebar

Note: All spacing uses Tailwind's default 4px scale. The dashboard CSS already defines these via utility classes.

## Color

Light mode:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #ffffff (white) | Card backgrounds, content area |
| Secondary (30%) | #f9fafb (gray-50) | Page background, table header rows, KPI icon containers |
| Accent (10%) | #465fff (brand-500) | Active nav items, chart primary color, CTA buttons, links |
| Success | #12b76a (success-500) | Positive trends (up arrows), completed status badges, active status |
| Warning | #f79009 (warning-500) | Pending status badges, caution indicators, ban reason highlight |
| Error | #f04438 (error-500) | Negative trends (down arrows), failed status, banned status, destructive actions |

Dark mode:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #1a2231 (gray-dark) | Card backgrounds |
| Secondary (30%) | #0c111d (gray-950) | Page background |
| Accent (10%) | #465fff (brand-500) | Same usage as light, with dark-aware variants (brand-400 for text) |

Accent reserved for:
- Active sidebar navigation items (bg-brand-50 / dark:bg-brand-500/[0.12])
- Chart primary series color
- CTA buttons (Mark as Paid, Send Reminder)
- Date range selector active tab
- Links in table rows (View, View Invoice)
- KPI trend up-arrow indicators on positive metrics

Destructive actions (error-500/error-600):
- Ban user confirmation
- Reject order

All colors are already defined in `globals.css` as Tailwind theme tokens (brand-*, gray-*, success-*, warning-*, error-*). No new color values needed.

## Interaction Patterns

### Date Range Selector
- 4 preset buttons: "Last 7 Days", "Last 30 Days", "Last 90 Days", "This Year"
- Active state: `bg-brand-50 text-brand-500 border-brand-500` (matches ChartTab pattern)
- Inactive state: `bg-white text-gray-500 border-gray-200`
- On click: updates URL searchParams `?range=7d|30d|90d|year`, triggers server component re-render
- Applies to both KPI trends and revenue chart simultaneously

### KPI Trend Indicators
- Up trend: `ArrowUpIcon` + green Badge + "+X.X%" text
- Down trend: `ArrowDownIcon` + red Badge + "-X.X%" text
- Flat: gray Badge + "0%" text
- Format: "+12.5% vs last month" (per D-03)
- Division-by-zero guard: show "+100%" if current > 0 and previous = 0, show "N/A" if both are 0

### Revenue Chart Adaptive Type
- 7 Days: bar chart, grouped by day, `borderRadius: 5`, `columnWidth: 50%`
- 30 Days: area chart, grouped by week, smooth curve, gradient fill
- 90 Days: bar chart, grouped by month, `borderRadius: 5`, `columnWidth: 50%`
- This Year: area chart, grouped by month, smooth curve, gradient fill
- Chart height: 310px (consistent with StatisticsChart reference)
- Colors: primary `#465FFF`, no secondary series
- Y-axis formatter: BDT locale string
- Tooltip: `{value} BDT`
- Font: `DM Sans, sans-serif`

### Invoice Filter Tabs
- Horizontal tab bar above table: All / Paid / Pending / Failed
- Active tab: `bg-brand-50 text-brand-500 border-brand-500`
- Count badge on each tab: `({count})` after label
- Filter applies to table data immediately (client-side or URL searchParams)

### Mark as Paid (Invoice Action)
- Visible only on pending orders
- Button style: primary (brand-500 bg, white text)
- On click: confirmation modal "Mark this order as paid? This will trigger license creation and send a confirmation email."
- On confirm: calls server action reusing `verifyOrder` logic (D-15)
- After success: `router.refresh()` to update table

### Send Payment Reminder
- Visible only on pending orders
- Button style: outline (border-gray-200, text-gray-700)
- On click: sends payment reminder email via server action
- Toast/feedback: "Payment reminder sent to {customer email}"
- Uses new `payment-reminder.ts` email template

### User Role Change
- Select dropdown on user detail page
- Options: customer, admin, support_staff, super_admin
- On change: confirmation modal "Change {userName}'s role from {current} to {new}?"
- On confirm: calls server action, validates role against enum, updates DB
- After success: `router.refresh()`

### User Ban/Activate Toggle
- Toggle button on user detail page
- If active: "Ban User" button (error/destructive style)
- If banned: "Activate User" button (success/outline style)
- On ban: modal with required ban reason textarea
- On activate: confirmation modal "Reactivate {userName}'s account?"
- On confirm: calls server action, updates banned + banReason fields

### CSV Export
- "Export CSV" button above data tables (orders, invoices, users, activity)
- Button style: outline with Download icon
- Client-side: generates CSV from currently visible/filtered data
- BOM prefix for Excel UTF-8 compatibility
- Filename format: `{report-type}-{YYYY-MM-DD}.csv`
- BDT currency formatting in amounts
- BD date format: `DD Mon YYYY`

### Activity Feed Event Types
Icon mapping per event type:

| Event | Icon | Icon Color | Background |
|-------|------|------------|------------|
| order.created | ShoppingCart | text-brand-500 | bg-brand-50 dark:bg-brand-500/15 |
| order.status_changed | RefreshCw | text-success-600 | bg-success-50 dark:bg-success-500/15 |
| license.created | Key | text-brand-500 | bg-brand-50 dark:bg-brand-500/15 |
| user.registered | UserPlus | text-success-600 | bg-success-50 dark:bg-success-500/15 |
| ticket.created | MessageSquare | text-warning-600 | bg-warning-50 dark:bg-warning-500/15 |
| user.banned | ShieldBan | text-error-600 | bg-error-50 dark:bg-error-500/15 |
| user.role_changed | UserCog | text-blue-light-600 | bg-blue-light-50 dark:bg-blue-light-500/15 |
| order.refunded | RotateCcw | text-error-600 | bg-error-50 dark:bg-error-500/15 |

Description format: Human-readable sentence using action + details fields from audit_logs. Examples:
- "New order placed" (order.created)
- "Order verified and confirmed" (order.status_changed, to=completed)
- "License key generated" (license.created)
- "New user signed up" (user.registered)
- "Support ticket opened" (ticket.created)
- "User banned: {reason}" (user.banned)
- "User role changed to admin" (user.role_changed)

Timestamp: `formatDistanceToNow(createdAt, { addSuffix: true })` from date-fns (consistent with NotificationDropdown)

### Admin Notifications (NotificationDropdown Extension)
- Extend existing NotificationDropdown to detect admin context
- When user role is admin/super_admin AND route is /admin/*: query admin-scoped notifications
- New notification types with icon mapping:

| Type | Icon | Color |
|------|------|-------|
| payment_failed | AlertTriangle | error |
| license_expiring | Clock | warning |
| new_signup | UserPlus | success |
| new_ticket | MessageSquare | warning |
| fraud_alert | ShieldAlert | error |

- Stored in existing notifications table with admin user ID
- Created proactively when relevant events occur (order failed, license expiring, new user, new ticket)
