- "use client"
- Uses `useSearchParams()` to read `plan` query param
- Import pricingTiers from `@/data/pricing.ts` and find matching tier by lowercase plan name
- If plan not found, show error: "Invalid plan selected. Please go back to pricing and select a plan." with link to `/pricing`
- State management with useState:
  - `selectedMethod: string | null`
  - `couponCode: string`
  - `appliedCoupon: { code: string; discount: number } | null`
  - `couponError: string | null`
  - `isSubmitting: boolean`
  - `vatInfo: { taxAmount: number; total: number; rate: number; mode: string } | null`
  - `paymentAccounts: Record<string, PaymentAccount>`
  - `submitError: string | null`
- On mount: call `calculateVAT(basePrice)` to get VAT, call `getPaymentAccounts()` to get payment details
- Layout per UI-SPEC: 2-column on desktop (grid grid-cols-1 lg:grid-cols-2 gap-8), stacks on mobile
- Left column: PageBreadcrumb ("Home > Checkout"), OrderSummary component
- Right column:
  - PaymentMethodGrid component
  - CouponInput component (below payment methods, with gap-4)
  - Conditional rendering based on selectedMethod:
    - If manual method (bkash/nagad/rocket/bank_transfer): PaymentInstructions + ManualPaymentForm
    - If ssl_commerz: "Pay with SSL Commerce" button that calls create-session API
- Plan price extraction from pricingTiers: parse the BDT price string to get integer amount. Use a helper: parse the `priceBDT` field (e.g., "≈ ৳2,150 BDT" -> extract digits -> 2150, or "≈ ৳3,000 BDT" -> 3000, or "≈ ৳8,000 BDT" -> 8000). Actually, define a helper map in the component:
```typescript
const planPrices: Record<string, number> = {
  starter: 2150,
  professional: 3000,
  agency: 8000,
};
```
  This is simpler and more reliable than parsing the formatted string. The pricing.ts data is the display layer; the integer amounts for orders come from this authoritative map.

- Coupon apply handler:
  - Call `validateCoupon(code, basePrice)` server action
  - On success: set appliedCoupon, recalculate total (basePrice - discount + vat)
  - On error: set couponError

- Manual order submit handler:
  - Call `createManualOrder` with FormData containing: plan, paymentMethod, paymentRef, couponCode, amount (server-validated), taxAmount, discountAmount
  - On success: redirect to `/dashboard/checkout/success?order=${result.orderId}`
  - On error: set submitError

- SSL Commerce handler:
  - POST to `/api/ssl-commerz/create-session` with JSON body: { plan, couponCode, discountAmount, taxAmount, totalAmount }
  - On success: `window.location.href = response.url` (redirect to SSL hosted page)
  - On error: show "Unable to connect to payment gateway" error per UI-SPEC
