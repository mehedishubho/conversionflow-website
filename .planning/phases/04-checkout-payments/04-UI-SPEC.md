## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, form group gaps |
| md | 16px | Default element spacing, card padding (p-4) |
| lg | 24px | Section padding, card body padding (p-6) |
| xl | 32px | Layout gaps between columns |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Payment method card grid gap: 12px (3 in Tailwind) -- tighter than standard to fit 5 cards
- Checkout column gap: 32px on desktop, 24px on tablet

## Page Layouts

### 1. Checkout Page (`/dashboard/checkout?plan=X`)

**Layout:** 2-column inside ComponentCard. Portal sidebar and header visible.

```
+-------------------------------------------+
| Breadcrumb: Home > Checkout               |
+-------------------------------------------+
| Order Summary (left)  | Payment (right)   |
| +-------------------+ | +--------------+  |
| | Plan: Professional| | | Select Pay   |  |
| | Base: 3,000 BDT   | | | Method:      |  |
| | VAT (15%): 450     | | | [bkash][nag] |  |
| | Coupon: -300       | | | [rock][ssl]  |  |
| | ──────────────     | | | [bank]       |  |
| | Total: 3,150 BDT   | | +--------------+  |
| |                    | | | Coupon Code  |  |
| | Coupon: SAVE10     | | | [input][Apply]|  |
| | [Remove]           | | +--------------+  |
| +-------------------+ | | Instructions  |  |
|                       | | (method-specific)|
|                       | | Transaction ID  |  |
|                       | | [input]         |  |
|                       | | [Submit Order]   |  |
|                       | +-----------------+  |
+-------------------------------------------+
```

**Responsive:** Stacks to single column below 768px. Order summary first, then payment section.

**States:**
- Initial: Plan loaded from URL param, no method selected, no coupon applied
- Method selected: Shows method-specific instructions + transaction ID input (manual) or redirect button (SSL)
- Coupon applied: Discount line appears in summary, "Remove" link replaces Apply button
- Submitting: Button shows spinner, disabled state
- Error: Inline error below the relevant input (red text, error-500)


## Interactive States

### Payment Method Card States

| State | Visual |
|-------|--------|
| Default | 1px border gray-300, white background, no left accent, method icon + name |
| Hover | Border gray-400, subtle shadow (shadow-theme-xs), cursor pointer |
| Selected | 2px border brand-500, 3px left border in method brand color, brand-50 background |
| Disabled (not configured) | Opacity 0.5, gray-200 border, "Not Available" text, cursor not-allowed |

### Button States

| State | Visual |
|-------|--------|
| Default | As per Button component (brand-500 bg for primary, ring outline for outline) |
| Hover | brand-600 bg for primary, gray-50 bg for outline |
| Loading | Spinner replaces text, disabled, brand-300 bg |
| Disabled | Opacity 0.5, cursor not-allowed |

### Order Summary Calculation Animation

When coupon is applied or removed, the discount line and total should update with a brief highlight animation:
- New line appears: slideDown + fadeIn (0.2s ease)
- Changed total: brief brand-100 background flash (0.3s)
