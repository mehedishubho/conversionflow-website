Three server actions, all with admin role check:

```typescript
const session = await auth.api.getSession({ headers: await headers() });

```

**1. `verifyOrder(orderId: string)` -- per D-09, LIC-01:**
- Role check (admin/super_admin only)
- Fetch order by ID, verify status is "pending"
- Update order status to "completed"
- Create audit log: `order.status_changed` with details { from: "pending", to: "completed" }
- Sync to central API: call `importOrderToCentral()` (or mock) with order data + user data
  - On success: update order.centralOrderId, user.centralUserId, create license record
  - On failure: order stays completed but no central mapping (pending_sync state per D-14)
- Return `{ success: true }` or `{ error: string }`

**2. `rejectOrder(orderId: string, reason: string)` -- per D-09, D-20:**
- Role check
- Fetch order by ID, verify status is "pending"
- Update order status to "failed" (rejected orders use "failed" status)
- Create audit log: `order.status_changed` with details { from: "pending", to: "failed", reason }
- Return `{ success: true }` or `{ error: string }`

**3. `issueRefund(orderId: string, reason?: string)` -- per D-20:**
- Role check
- Fetch order by ID, verify status is "completed"
- Update order status to "refunded"
- If order has a linked license, update license status to "revoked"
- Create audit log: `order.status_changed` with details { from: "completed", to: "refunded", reason }
- Return `{ success: true }` or `{ error: string }`
